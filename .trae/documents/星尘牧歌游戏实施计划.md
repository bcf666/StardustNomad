# 《星尘牧歌》游戏设计实施计划 (修订版)

## 一、项目概述

**项目名称**: StardustNomad / 星尘牧歌
**项目类型**: 太空沙盒建造游戏
**视觉风格**: 体素/Minecraft风格
**技术方案**: Three.js 3D渲染 + 原生JavaScript游戏引擎
**目标**: 实现完整设计文档中的所有核心系统

---

## 二、技术架构

### 2.1 技术栈
- **渲染引擎**: Three.js (WebGL)
- **体素系统**: 简化版体素引擎，支持大规模世界
- **构建工具**: 纯前端静态资源，无需构建
- **存储**: IndexedDB (Chunk差异存档)

### 2.2 核心设计决策

#### 星球实现方案 (方案A - 简化版)
- 太空中的"星球"是低多边形球体装饰
- 玩家靠近到一定距离时，触发"进入大气层"过渡效果
- 实际游戏区域是独立的体素平面世界
- 大幅降低技术复杂度，优先保证核心玩法

#### Chunk规格
- **尺寸**: 32x32x32 (高度可扩展到64)
- **内存优化**: 面可见性裁剪 + 网格合并
- **加载策略**: 玩家周围12个Chunk常驻，超出范围异步卸载

#### 飞船系统简化 (MVP阶段)
- 飞船仅支持外部建造和驾驶舱视角
- 玩家不能进入飞船内部自由行走
- 内部改装通过专用建造界面完成
- 后续版本再考虑内部行走系统

## 三、两大开发阶段

### 阶段一：MVP - 单一星球建造与飞行 (核心目标)

**目标**: 2-3周内完成可玩闭环

1. 体素渲染 + 无限平面世界 + 基础地形
2. 第一人称角色控制 (行走、跳跃、挖掘、放置)
3. 飞船作为可载具实体 (不可进入内部)
4. 飞船物理飞行 (推力/质量/悬停)
5. 采矿激光采集资源
6. 星球表面基础建造 + 简单矿机
7. 本地存档 (IndexedDB脏区块存储)

**验收**: 这个闭环跑通并证明乐趣后，再进行阶段二

### 阶段二：扩展 - 星际探索与自动化

1. 多星球/星系生成和跃迁
2. 飞船内部建造和步行 (可选)
3. 传送带、冶炼厂、物流站等自动化流水线
4. 科技树完整系统
5. 派系与战斗系统

### 4.5 飞船系统

**文件**: `src/ship/ship.js`
**文件**: `src/ship/ship-builder.js`
**文件**: `src/ship/physics.js`

核心功能:
- 飞船数据结构 (网格位置 + 方块类型)
- 飞行物理:
  - 推力 = 引擎方块数量 × 引擎功率
  - 质量 = 所有方块质量之和
  - 速度 = 推力 / 质量 × 时间
  - 大气层阻力 (高度越高阻力越小)
- 飞船模块类型:

```javascript
ShipModule = {
  // 框架
  FRAME_LIGHT: { mass: 1, strength: 1 },
  FRAME_HEAVY: { mass: 3, strength: 3 },
  // 推进器
  ENGINE_CHEMICAL: { thrust: 10, power: 5, mass: 2 },
  ENGINE_ION: { thrust: 5, power: 3, mass: 1 },
  ENGINE_WARP: { thrust: 0, power: 0, mass: 5, enablesWarp: true },
  // 能源
  REACTOR_SOLAR: { power: 2, mass: 1 },
  REACTOR_FISSION: { power: 10, mass: 4 },
  REACTOR_FUSION: { power: 25, mass: 6 },
  // 功能舱
  COCKPIT: { mass: 2 },
  CARGO_BAY: { mass: 1, capacity: 1000 },
  MINING_LASER: { mass: 3, miningPower: 5 },
  // 护盾
  SHIELD_BASIC: { shieldPower: 20, powerDrain: 2, mass: 2 }
}
```

飞船建造模式:
- 网格对齐的体素放置 (类似Space Engineers)
- 预览放置效果 (半透明)
- 碰撞检测 (与地表、其他物体)
- 重心计算 (动态)
- 飞行动力学实时更新

### 4.6 多方块结构识别

**文件**: `src/ship/module-detector.js`

功能:
- 模板匹配识别功能模块 (如"L型引擎=2x2x2框架+后方推进器")
- 功能舱室识别 (驾驶舱必须是开放结构)
- 特殊结构检测 (曲速引擎需要特定形状)

### 4.7 资源与科技系统

科技树结构 (MVP阶段简化):
- **采矿科技**: 基础采矿 → 高效采矿激光 → 自动采矿机

## 七、存档系统

**文件**: `src/save/save-manager.js`

存储策略 (IndexedDB):
- **Chunk差异存储**: 只存储玩家修改过的Chunk (脏区块)
- **未修改Chunk**: 由程序根据种子重新生成
- **存档内容**:
  - 飞船结构
  - 基地结构
  - 资源库存
  - 科技进度
  - 星球探索状态
  - 玩家位置

```javascript
SaveData = {
  version: 1,
  seed: 12345,
  player: { position, inventory, stats },
  ship: { blocks: [...], modules: [...] },
  bases: [{ position, blocks: [...] }],
  modifiedChunks: [{ x, y, z, blocks: [...] }],
  techProgress: [...]
}
```
