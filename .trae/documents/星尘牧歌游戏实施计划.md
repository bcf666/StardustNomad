# 《星尘牧歌》游戏设计实施计划 v2.0

## 一、项目概述

**项目名称**: StardustNomad / 星尘牧歌
**项目类型**: 太空沙盒建造游戏
**视觉风格**: 体素/Minecraft风格
**技术方案**: Three.js 3D渲染 + 原生JavaScript游戏引擎
**开发策略**: 先跑通核心可玩闭环，再逐步扩展

## 二、技术架构

### 2.1 技术栈
- **渲染引擎**: Three.js (WebGL)
- **体素系统**: Chunk 32x32x32 区块，动态加载
- **物理**: 简化的刚体物理 + 自定义飞行力学
- **存储**: IndexedDB (Chunk差异存储)
- **构建**: 纯前端静态资源

### 2.2 星球实现方案：方案A（简化）
- 太空中的星球是**低多边形球体装饰**（视觉）
- 玩家靠近到一定距离，触发"进入大气层"过渡效果
- 传送到该星球对应的**平面体素世界**
- 极大降低技术复杂度，保留探索沉浸感

### 2.3 飞船系统：简化方案
- **初期不做内部行走**，飞船作为整体刚体移动
- 飞船通过**建造界面**进行模块化拼装
- 玩家在**驾驶舱视角**控制飞船
- 飞船改装 = 在网格编辑器中编辑方块
- 内部装修延后到扩展阶段

### 3.2 玩家系统

**文件**: `src/player/player.js`

玩家状态:
```javascript
PlayerState = {
  position: Vector3,      // 世界坐标
  velocity: Vector3,      // 移动速度
  health: 100,             // 生命值
  oxygen: 100,             // 氧气（太空/水下）
  temperature: 20,         // 体温
  armorRating: 0,         // 护甲等级
  inventory: Inventory,   // 背包系统
  hotbar: Hotbar,         // 快捷栏(9格)
  equipped: ToolType      // 当前手持工具
}
```

状态机制:
- 生命值: 跌到0 = 复活到最近存档点
- 氧气: 太空/水下持续消耗，0时持续掉血
- 温度: 过冷/过热持续掉血，需要宇航服抗性

**文件**: `src/player/inventory.js`
- 物品ID统一系统
- 堆叠逻辑 (同类型最多64个)
- 快捷栏管理
- 物品转移界面 (容器 ↔ 玩家)

### 3.3 飞船系统

**文件**: `src/ship/ship.js`
- 飞船数据结构: `Map<"x,y,z", BlockType>` 网格字典
- 飞船性能计算:
  ```
  总质量 = Σ(各方块质量)
  总推力 = Σ(引擎方块推力)
  最大速度 = 总推力 / 总质量
  能源上限 = Σ(能源核心输出)
  ```

**文件**: `src/ship/ship-modules.js`

飞船模块定义 (多方块结构识别):
```javascript
ShipModule = {
  // 引擎模块 (3x1x1 连续引擎方块)
  CHEMICAL_ENGINE: { size: [3,1,1], thrust: 100, power: 10 },
  ION_ENGINE: { size: [3,1,1], thrust: 50, power: 5, efficient: true },
  WARP_ENGINE: { size: [5,2,2], thrust: 0, power: 0, warpCapable: true },

  // 能源核心 (中心1x1x1 + 框架环绕)
  FISSION_REACTOR: { power: 1000 },
  FUSION_REACTOR: { power: 5000 },
  ANTIMATTER_REACTOR: { power: 20000 },

  // 驾驶舱 (必选, 1x1x1)
  COCKPIT: { required: true },

  // 货舱 (1x2x1)
  CARGO: { capacity: 1000 },

  // 采矿激光 (2x1x1)
  MINING_LASER: { powerCost: 50, miningSpeed: 5 },

  // 护盾发生器 (2x2x2)
  SHIELD_GENERATOR: { shieldPower: 200, powerCost: 30 }
}
```

多方块识别算法:
- 扫描Ship网格，寻找已知模块的"锚点方块"
- 从锚点向指定方向延伸，检查尺寸和材质是否匹配
- 匹配成功则注册该模块及其提供的属性加成

**文件**: `src/ship/ship-physics.js`
- 推力/质量/阻力物理模型
- 大气层内外阻力变化
- 重力影响 (星球引力场)
- 碰撞检测 (飞船 vs 地形)

**文件**: `src/ship/ship-builder.js`
- 网格编辑器界面
- 放置/删除方块
- 模块预览
- 重心可视化
- 保存/加载飞船配置

## 六、关键算法

### 6.1 多方块模块识别算法

```javascript
function detectModule(shipGrid, x, y, z, moduleDef) {
  const [dx, dy, dz] = moduleDef.size;
  const blocks = [];

  for (let i = 0; i < dx; i++) {
    for (let j = 0; j < dy; j++) {
      for (let k = 0; k < dz; k++) {
        const block = shipGrid.get(`${x+i},${y+j},${z+k}`);
        if (block !== moduleDef.blockType) return null;
        blocks.push({x: x+i, y: y+j, z: z+k});
      }
    }
  }
  return { type: moduleDef.type, blocks, power: moduleDef.power };
}
```

### 6.2 Chunk差异存储

```javascript
// 保存时：只存储被修改的Chunk
const modifiedChunks = getModifiedChunks(); // 玩家放置/删除的Chunk
for (const chunk of modifiedChunks) {
  saveToIDB(chunk.key, serializeChunk(chunk));
}

// 加载时：检查本地Chunk是否存在
// 不存在则用种子重新生成
const localChunk = await loadFromIDB(chunkKey);
if (localChunk) {
  restoreChunk(localChunk);
} else {
  generateFromSeed(chunkKey);
}
```
